# 格斗机器人全套代码融合实现计划

本项目旨在将已有的**软启动**、**台下自动登台**、**台上巡台与寻敌进攻**四个模块有机融合，并遵循用户补充的物理场景要求，设计一套具有闭环自愈能力的鲁棒状态机。

---

## 优化逻辑与物理方案设计

为了响应您的补充说明，我们将采用以下设计方案：

### 1. 灰度渐变检测算法（区分启动区与擂台边界）
* **物理特征：**
  * 擂台边界：黑色与白色之间是**渐变过渡**的（灰度值从大到小或从小到大，平滑穿过 `[50, 200]` 区间）。
  * 铺台下启动区（蓝色/黄色）：与周围黑色地板是**瞬时跳变**的（无渐变过程）。
* **实现方案：**
  * 在状态机运行过程中，我们使用一个连续确认计数器 `grey_gradual_cnt[4]` 监测 4 路底盘灰度值。
  * 每当灰度值落在 `[50.0f, 200.0f]` 区间时，对应的计数器自增；若脱离该区间，计数器减 1（直至归零）。
  * 只有当任意一路灰度传感器的计数器达到 `GREY_GRADUAL_FILTER_CNT`（暂定为 6 次，即持续时间超过 120ms）时，才触发 `grey_gradual_success = 1`。这样可以完美过滤掉台下蓝色/黄色启动区的瞬时跳变，以及车身碰撞产生的瞬时电信号抖动。

### 2. 多阶段确认流程（初步确认 -> 执行程序 -> 最终确认）
当小车执行冲台（初始冲台或自动登台）时：
* **初步确认：** 实时监控 `Pitch` 和 `Roll`。一旦检测到倾斜（`> 12` 或 `< -12`），随后恢复到水平面（均在 `[-12, 12]` 内），则**初步判定成功**。
* **执行程序：** 此时立即停止冲台电机，切换状态至台上融合模式（`ROBOT_RUNNING`），让小车开始执行巡台与寻敌程序。
* **最终确认：** 在小车运行的同时，检查在刚才移动/冲台的过程中，是否触发过上述的 `grey_gradual_success == 1`（灰度渐变成功）。
  * 如果触发过，则**确定登台成功**，继续保持在台上融合模式。
  * 如果在冲台结束或进入台上模式后的 1 秒内（超时保护）都**未检测到灰度渐变**，则判定为**登台失败**（只是在台下撞到了土包或障碍物导致车身倾斜）。此时小车会退出融合模式，自动重新执行台下登台程序。

### 3. 台上运行时的掉台检测
* 当小车在台上常规运行（`ROBOT_RUNNING`）时，我们同样监控 `Pitch` 和 `Roll`。
* 一旦车身倾斜且恢复水平，且**在此期间检测到灰度渐变**，则**最终确定掉下擂台**。
* 确定掉台后，小车首先旋转对齐至最近的 90° 倍数方向（0°、±90°、±180°），随后自动开始台下扫描与顶墙登台动作，实现全自动闭环恢复。

## Verification Plan

### Manual Verification
1. **启动区软启动测试：** 将小车置于蓝色或黄色启动区，软启动后，小车全速后退。
   * 验证：跨越启动区边界（突变）时，小车**不应该**误报灰度渐变成功。
   * 验证：当小车越过擂台坡道（倾斜且恢复，且有灰度渐变）时，顺利进入台上巡台寻敌模式。
2. **登台失败退回测试：** 阻挡小车使其无法登台，或者在平地上模拟倾斜（不触发灰度渐变）。
   * 验证：小车倾斜恢复后进入台上模式，但由于未检测到灰度渐变，1秒内自动退回台下自动登台程序（开始旋转扫描）。
3. **掉台自愈测试：** 在小车巡台时，手动将其推下擂台（或使其自主掉落）。
   * 验证：小车检测到掉台后，立即停止巡台，转向至最近 of 90° 方向，然后重新自动顶墙并爬回擂台。
