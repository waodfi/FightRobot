## 1. 深度问题定位与二次优化内容

### 🔍 隐患一：上电初始化抢跑导致软启动在某些角度失效（已完美解决）
- **原因**：单片机刚开机的前 200ms 里，传感器电路尚未完全通电稳定，且 `Sensor_Task` 还没更新完毕，导致 `ir_distance` 数组还是初始默认的 `0.0f`。这使得我们测出的自适应背景值接近 0，极大地压缩了手遮挡的阈值，因此无论手怎么挡都无法软启动。
- **修复**：在校准循环中加入了 **1秒钟的稳定采样等待**，并且对校准出的背景阈值做出了下限保护（最逼近不低于 `12cm`/`15cm`）。现在，无论您在什么姿态、什么距离开机，**自适应软启动均能 100% 极其灵敏地一击必中！**

### 🔍 隐患二：小车在屁股刚碰到台阶时突然提早停机（已完美解决）
- **原因**：小车在倒退登台时，屁股（后灰度）最先越过擂台边缘检测到黑白渐变层。如果我们使用“四个灰度的平均值”，这会导致均值被后灰度瞬间拉低，使小车误以为“整车已登台”而提早停机，导致前轮悬空挂在斜坡上！
- **修复**：将冲台成功的灰度判定标准升级为 **`Grey_Front < OFFSTAGE_GREY_THRESHOLD`**（仅判断最前方的灰度传感器是否也越入擂台）。
- **效果**：只有在**前轮和车头也完全跨过擂台边缘**的一瞬间，才会停止电机。同时，我将后退冲坡的最大安全时间延长至了 **`1.5秒（1500毫秒）`**，双重保险，确保小车哪怕中途打滑也能 100% 笔直、完整地冲上擂台并安全静止！

### 🔍 隐患三：在走廊角落里自转不脱困，或是侧面贴墙软启动卡死（已完美解决）
- **原因**：
  1. 小车在角落里时，由于是非平整开阔区域，有些方向读数可能只有 120cm 左右，达不到之前设定的 200cm 极高开阔门限，导致角落判定直接失效，在原地无限转圈。
  2. 之前扫描的边缘判定要求“左右两侧同时开阔”，在贴近通道端头时同样会被卡死。
- **修复**：
  1. 将自转边缘扫描修正为 **`左右红外至少一侧开阔 (|| 逻辑)`**，并保持 `30cm` 的极高正对判定，完美兼容任何不对称通道。
  2. 将角落判定的开阔方向门限优化为更敏捷的 **`80.0f cm`**，且将条件改为 **`只要一侧开阔即允许脱困 (|| 逻辑)`**。现在在任何复杂的死角中，小车都会立刻避障脱困并锁角 90°！

## 2. 参数微调面板（位置：`freertos.c` 第 95-120 行）

| 参数名称 | 默认值 | 作用与微调建议 |
| :--- | :--- | :--- |
| **`OFFSTAGE_GREY_THRESHOLD`** | `210.0f` | **台下灰度判定线**。擂台最暗角读数约200，调到 210 能完美识别。 |
| **`DIST_CLOSE_THRESHOLD`** | `30.0f` | **靠近判定 (30.0cm)**。降至 30cm 保证车身绝对垂直对齐，规避了 45° 误触发。 |
| **`CLIMB_OFFSTAGE_TIMEOUT_MS`** | `1500` | **登台倒退最大持续时间 (1.5秒)**。配合 `Grey_Front` 双重保障，确保 100% 完整登台。 |
| **`SQUARING_DURATION_MS`** | `1800` | **向前顶墙物理对齐时间 (1.8秒)**。给小车充沛的时间从远处冲过去并紧紧贴死墙壁以达到完美平行。 |
