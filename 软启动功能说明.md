# 软启动功能（Soft Start）实现说明与测试指南

我们已经成功在您的 STM32H7 战斗机器人项目中实现了**软启动与冲上擂台**的核心逻辑。下面为您梳理代码结构、工作原理以及测试、调参方法。

---

## 1. 代码修改概要

所有的修改均在主控制任务文件 [freertos.c](file:///d:/Project/STM32_Project/FightRobot/Core/Src/freertos.c) 中完成：

1. **参数与变量定义（在 `USER CODE Variables` 区间）：**
   - **`SOFT_START_BLOCK_DIST (35.0f cm)`**：当左右红外测距传感器的检测距离均**小于**此值时，判定为“用手挡住”（实测挡手时数据通常降到 23~28cm 之间）。
   - **`SOFT_START_RELEASE_DIST (45.0f cm)`**：当左右红外测距传感器的检测距离均**大于**此值时，判定为“手拿开”（实测无遮挡时数据通常为 190~240cm 之间）。
   - **`SOFT_START_CONFIRM_CNT (5次)`**：连续确认的防抖次数（5次 × 20ms = 100ms），防止噪声误触发。
   - **`CLIMB_DURATION_MS (2000ms)`**：全速倒车冲上擂台的持续时间（2秒）。
   - **`CLIMB_SPEED (-100)`**：倒车速度（-100 表示全速后退）。
   - **状态机枚举 `RobotState_e` 与变量 `robot_state`**：管理机器人的状态切换（`WAITING` 等待 -> `ARMED` 就绪 -> `CLIMBING` 冲台 -> `RUNNING` 正常巡逻/上位机模式）。

2. **状态机控制逻辑（在 `StartMotion_Task` 运动任务中）：**
   - **初始化延时**：在任务主循环开始前加入了 `osDelay(1000)`，等待红外测距传感器上电稳定。
   - **状态机拦截**：在 `for(;;)` 循环的最开始，通过 `if (robot_state != ROBOT_RUNNING)` 块拦截了正常的上位机控制逻辑。只有在机器人顺利完成“等待遮挡 -> 松开 -> 全速倒车上台”后，才会切换到 `ROBOT_RUNNING` 并交出控制权。

---

## 2. 状态机跳转与动作细节

机器人的状态变化如下：

- **等待阶段（`ROBOT_WAITING`）**：
  - 电机静止（输出速度 0）。
  - 不断检测 `IR_Distance_L`（左侧红外）与 `IR_Distance_R`（右侧红外）。
- **就绪阶段（`ROBOT_ARMED`）**：
  - 电机静止（输出速度 0）。
  - 双手继续挡住传感器时保持此状态，一旦双手同时移开，触发倒车。
- **冲台阶段（`ROBOT_CLIMBING`）**：
  - 启动电机全速后退（输出 `Motor_Control(0, -100)`）。
  - 保持后退 2000 毫秒（2 秒）。
- **运行阶段（`ROBOT_RUNNING`）**：
  - 电机暂时静止（输出速度 0），释放控制权。
  - 之后机器人会正常等待上位机上线或执行自动控制逻辑。

## 3. 串口调试信息

为了让您能够非常直观地了解机器人当前的运行状态，我们在各关键节点添加了中文串口打印信息。您可以将电脑通过串口线（通常为 USART1，波特率 115200）连接到机器人板子上，查看如下输出：

1. **上电时：**
   ```text
   [SoftStart] System initialized. Waiting 1s for sensors to stabilize...
   [SoftStart] Waiting for soft start trigger. Please cover BOTH left and right IR sensors with your hands.
   ```
2. **用手挡住左右红外传感器后：**
   ```text
   [SoftStart] Both IR sensors covered! State -> ROBOT_ARMED. Release both hands to launch!
   ```
3. **把手拿开的一瞬间：**
   ```text
   [SoftStart] Hands released! State -> ROBOT_CLIMBING. Full speed reverse for 2000 ms!
   ```
4. **全速倒车2秒结束后：**
   ```text
   [SoftStart] Climbing complete! State -> ROBOT_RUNNING. Handover to normal control.
   ```

## 4. 调试与参数微调指南

作为机械专业的同学，您在实物测试中可能会遇到传感器响应、爬坡时间或场地摩擦力带来的影响。您可以重点微调 `freertos.c` 中第 96-100 行的几个核心宏定义：

1. **手遮挡判定距离：`SOFT_START_BLOCK_DIST`**
   - **当前设定**：`35.0f` (35cm)
   - **调整建议**：如果手放得太近才触发，可以调大（如 `40.0f`）；如果手还没怎么靠近就误触发，可以调小（如 `30.0f`）。

2. **手松开判定距离：`SOFT_START_RELEASE_DIST`**
   - **当前设定**：`45.0f` (45cm)
   - **调整建议**：通常只要明显大于遮挡距离即可。如果松手后反应较慢，可以尝试调小该值（如 `40.0f`，但须确保大于 `SOFT_START_BLOCK_DIST`）。

3. **倒车冲台时间：`CLIMB_DURATION_MS`**
   - **默认值**：`2000` (2秒)
   - **调整建议**：极关键！如果机器人冲上 6cm 擂台后还在疯狂原地倒车，说明时间太长，可以减小（如 `1500`）；如果还没爬上擂台就停了，说明时间不够，可以增大（如 `2500`）。

4. **倒车速度：`CLIMB_SPEED`**
   - **默认值**：`-100` (全速倒车)
   - **调整建议**：`-100` 是爆发力最强的设置。如果车速过快导致轮子打滑严重，可以适当降低速度绝对值（如 `-80`）。
